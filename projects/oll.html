Ovio - Open Impact
Open Law Library is a non-profit dedicated to making the law available to all in human- and computer-readable formats. We have built the Open Law Platform to empower governments to easily and rapidly publish their laws in open and free digital-native formats.

Core Challenge

We are taking the tools and techniques developed for programmers to publish software code - IDEs, linters, compilers, continuous integration/deployment, VCS, etc - and adapting and transforming them for use by lawyers to publish legal codes. We have two major challenges: (1) As our “code base” of laws grows (over 1.2m “LoC” and growing), we need to constantly improve our tooling to draft, annotate, navigate, compile, and understand the laws published on our platform; and (2) because we are working with official laws, security, auditability, and preservation are vital.

Projects

  • Cryptographic Authentication of Laws on Decadal timescale Open Law Library publishes all laws as GIT repositories, and these laws need to be cryptographically authenticatable. GIT signing has a number of issues. We are building a tool called The Archive Framework. We are looking for security experts who are interested in helping us build, secure, test, and/or audit this system. This project will make it possible for governments to easily publish versioned laws. This is a vital prerequisite for digitally publishing official laws.

  • DevOps Improvements We launched websites for our first two partner jurisdictions in 2018. We plan to launch for at least 10 more this year. Each partner’s website is customized. We host everything on AWS, and use Terraform/Packer/Ansible to create similar-but-not-identical environments for each partner. We are not DevOps people, but have been teaching ourselves. It would be incredibly helpful to have a volunteer with deep DevOps experience who could review our stack, help us incrementally improve our architecture design, and then help us implement those improvements.This will make it possible for us to not only rapidly onboard new jurisdictions (and thus publish more laws in open, accessible formats), but also ensure that we serve the needs of our existing clients.

  • ML/AI for automated lawyer assistance Each law can be thought of as a change set. We would like to use ML/AI to create a model to, given a location in the document, predict the codification instruction that describes that location of the document. Since These codification instructions need to be 100% correct, we would use this model to provide autocomplete within the IDE that would then be reviewed immediately by the attorney. This system will reduce the time it takes attorneys to perform basic codification tasks, freeing their time to perform higher-value annotations, improving the usefulness of the open laws published on the platform.

Current Tech Stack

  • DevOps: AWS; terraform, packer, ansible
  • Drafting IDE (Word plugin for lawyers) - C# VSTO; React; flask
  • Codification IDE (language server/ VS Code client) - Python (pygls); typescript
  • Codification engine and static site/document generator - Python/lxml
  • Dynamic website components: Vue; Python (django); OpenResty
  • Authentication of laws: Python (TUF, TAF)
